### 🔒 发布前优化
- 设置面板新增**隐私说明**（数据仅存本地、不上传服务器）
- API Key 区域新增 DeepSeek 获取链接引导
- **CDN 降级**：lz-string/qrcode/html5-qrcode 加载失败时友好提示替代方案
- SW 缓存版本管理（`CACHE_VERSION`），每次部署强制刷新
- 顶部栏新增 **🔄 强制刷新按钮**（一键清缓存+注销SW+重载）

### 🐛 导航衔接修复
- **备菜→做菜悬浮窗闪现**：移除按钮处理器中多余的 `stopTimer()`
- **做菜页退回按钮无效**：`goPrep()` 从 `goScreen('prep')` → `history.back()` → 最终用 `goScreen('prep')` 直接跳转
- **做菜第一步底部按钮**：从灰色不可点的「上一步」改为可点的「← 返回备菜」
- 根因：`stopSpeak()` 里 `speechSynthesis.cancel()` 无保护，抛错后 `goScreen` 永远不执行 → 所有 try/catch 加固

### 🎵 语音系统 — Android 攻坚战
- **结论**：Android 国产浏览器（微信、厂商自带）**不支持** `speechSynthesis` API
- 音色面板从「加载音色列表」改为「实测播报」（▶ 测试播报按钮）
- `getVoiceList()`/`openVoiceSelector()`/`speakStep()` 全程 try/catch 保护
- 启动时自动检测语音是否可用，不可用时弹一次性提示
- 浏览器识别：微信/QQ/UC/厂商自带 → 分别给出不同引导
- **唯一解决方案**：用 Google Chrome 浏览器，或重写为微信小程序

### 📱 移动端 JS 陷阱总结
- `![]` 是 `false`（空数组检查必须用 `.length`）
- 模板字符串 `` `${}` `` 和展开运算符 `[...arr]` 旧手机不支持
- CSS 变量设 inline style（`btn.style.background = 'var(--)'`）移动端不解析
- `scrollIntoView` 在隐藏元素上可能抛错
- `addEventListener` 在动态启用的 `disabled` 按钮上不可靠 → 改用 `onclick`

### 🤖 下一步计划
- **微信小程序版**：用微信同声传译插件替代 `speechSynthesis`，解决语音兼容性
- 需重写：`.wxml` 页面结构、`.wxss` 样式、微信 API 数据存储
- 大部分 JS 逻辑可复用

### 📁 架构重构：数据与 UI 分离

- **之前**：4200+ 行单文件，51 道菜谱数据嵌入 `index.html`
- **之后**：`recipes.js`（1621 行，纯菜谱数据）+ `index.html`（2740 行，纯 UI 逻辑）
- `index.html` 通过 `<script src="recipes.js"></script>` 加载数据
- 新增 `loadRecipes()` 自动合并逻辑：localStorage 旧数据缺 `tags` 等新字段时，自动从 DEFAULT_RECIPES 补齐

### ⚙️ 管理后台重构

- **设置面板**：备菜方式、计时器模式、API Key、音色管理整合为顶部折叠面板

### 🏷️ 标签系统

首页标签栏可筛选，支持与搜索叠加。标签栏和排序栏添加桌面端 ◀ ▶ 滚动箭头。

### 🎵 语音系统

- **永久移除慧慧**：`getVoiceList()` 硬编码过滤 `huihui`，不受 localStorage 影响
- **晓晓设为默认**：`getSelectedVoice()` 优先搜索 `xiaoxiao`
- **播报打断**：`stopSpeak()` 无条件 `cancel()`，切换步骤时立刻停旧播新
- **手机端语音加载**：空列表时自动发无声 `speak()` 唤醒引擎，`voiceschanged` 自动刷新面板

### ⏱ 计时器系统

- **餐铃音效**：双频（1800Hz+2400Hz）正弦波模拟金属铃"叮"声，0.8s 指数衰减
- **自动模式**：切换步骤后 300ms 自动开始倒计时（默认开启）
- **手动/自动切换**：管理后台可切换，`localStorage` 持久化
- **时间到提示**：铃铛 + 语音 + 计时器区域闪烁（移除阻断式 `alert()`）
- **悬浮倒计时窗**：离开做菜页后底部显示橙色药丸浮窗，三种状态（运行中/已暂停/时间到），点击跳回做菜页
- **计时器持续运行**：离开做菜页/锁屏不暂停，后台继续倒数

### 📱 移动端适配

- **浏览器退回键**：`history.pushState` + `popstate` 监听，退回回到上一页而非退出 App
- **备菜方式切换**：修复缺失的 `togglePrepMode()` 函数

### 📡 跨设备同步

- **导出备份**：11 项 localStorage 数据打包为 JSON 文件下载（API Key 脱敏确认）
- **导入恢复**：文件选择 → 验证版本号 → 写入 localStorage → 刷新页面
- **二维码分享**：LZString 压缩 → QRCode.js 生成 → 另一设备扫码导入
- **扫码导入**：html5-qrcode 摄像头扫描 → 解析 → 恢复 → 自动关窗（失败可关闭）
- **版本校验**：未来版本数据拒绝导入
- **多标签同步**：`storage` 事件监听，标签页间自动同步菜谱/评分/设置
- **localStorage 配额保护**：`saveRecipes()` 捕获 `QuotaExceededError` 弹窗提示

### 🛡️ 边界情况处理

- `localStorage` 写满时弹窗提示而非静默失败
- 导入数据时版本号校验（v > 1 拒绝）
- 所有 `tags` 访问用 `(r.tags || [])` 容错
- 页面隐藏时计时器继续运行（用户反馈后才改了回来）

### 🔧 PWA 支持

新增文件：
- `manifest.json`：App 名、图标、全屏模式、主题色 `#f97316`
- `sw.js`：安装时预缓存核心文件，网络优先 + 3s 超时回退缓存，CDN 资源缓存优先

### 💬 产品策略讨论

- **差异化**：内容壁垒（人工验证菜谱）> 用户数据壁垒 > 品牌信任。别防抄袭，让抄袭成本 > 原创成本
- **出海**：海外中餐教学是蓝海，付费意愿高。需英文本地化 + 食材替代方案。国内测内容，海外挣钱
- **平台测试**：建议先上 GitHub Pages（零成本快速验证），有初步反馈再考虑微信小程序或海外版本

### 🎨 App 图标设计

- **旧图标**：`manifest.json` 中使用的 emoji SVG（🍳），不够专业
- **设计过程**：
  1. 评估多个 AI 图标工具：Midjourney（最佳质感）、Ideogram（文字渲染）、Flux（画质）、Adobe Firefly（商用安全）、DALL·E 3（上手快）→ 选择 **Midjourney**
  2. 原始概念：「手掌形状的锅 + 锅顶厨师帽」→ 一语双关但过于复杂，缩小后辨识度差，放弃
  3. 最终方案：「橙色圆底 + 白色厨师帽剪影 + 锅铲」→ 极简、2色、缩小到 64px 仍清晰
- **MJ Prompt**：
  ```
  A super minimal app icon, solid warm orange rounded square background,
  a pure white chef hat silhouette centered, a simple white spatula shape
  inside the hat, only 2 colors total, bold thick shapes, no details no
  texture no shadows no gradients, flat vector icon, iOS app icon style
  --style raw --s 50 --ar 1:1
  ```
- **选定文件**：`icon-384.png`（384×384，橙底白色厨师帽+锅铲剪影）
- **技术改动**：
  - `manifest.json`：`icons` 从 emoji SVG 改为 `icon-384.png`
  - `index.html`：新增 `<link rel="apple-touch-icon" href="icon-384.png">`
- **后续建议**：可放大到 512×512 以获得最佳 PWA 兼容性
